## Security Policy

### Supported versions

This project currently maintains the latest version on the `main` branch.

### Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Instead, report vulnerabilities privately to:

- GitHub username: `maxi720`
- Preferred channel: GitHub private contact (or repository owner contact method)

When reporting, include:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested fix (if available)

### Response expectations

- Initial response target: within 7 days
- Status updates: provided as investigation progresses
- Fix timeline: depends on severity and complexity

### Scope

Security reports are especially relevant for:

- File upload and parsing behavior
- Path handling and file storage
- Input validation and error handling

Thank you for helping keep SkyMonkey secure.
